# Contributing to Arict Web V2

First off, thank you for considering contributing to Arict Web V2! It's people like you that make open source such a great community.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create an issue on GitHub with:
1. A clear and descriptive title.
2. Exact steps to reproduce the problem.
3. What you expected to happen vs what actually happened.

### Suggesting Enhancements
Have an idea for a cool new feature or a design upgrade? 
1. Open an issue describing the feature.
2. Explain why this enhancement would be useful to most users.
3. If you have a mockup or code snippet, include it!

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code lints properly.
5. Issue that pull request!

## Styleguide
- Use modern React functional components and hooks.
- Follow the existing Dark Glassmorphism CSS patterns in `src/index.css`.
- Keep components modular and reusable.
